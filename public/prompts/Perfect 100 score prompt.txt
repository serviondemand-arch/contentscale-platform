### 5. MOBILE OPTIMIZATION (PERFECT 4/4)

**MOBILE-FIRST IMPERATIVE:**

As of November 2024, Google uses ONLY mobile version for indexing. 
Desktop version is irrelevant for rankings.

**CRITICAL REQUIREMENTS:**

**A. RESPONSIVE DESIGN:**

**Viewport Meta Tag (Required):**
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**Responsive Breakpoints:**
```css
/* Mobile first approach */
/* Base styles for mobile (320px+) */

@media (min-width: 768px) {
  /* Tablet styles */
}

@media (min-width: 1024px) {
  /* Desktop styles */
}
```

**Test on:**
- iPhone SE (375x667)
- iPhone 12/13/14 (390x844)
- Samsung Galaxy (412x915)
- iPad (768x1024)
- Desktop (1920x1080)

**B. TYPOGRAPHY:**

**Minimum Font Sizes:**
```css
body {
  font-size: 16px; /* NEVER smaller */
  line-height: 1.5; /* Minimum */
}

h1 { font-size: 28px; } /* Mobile */
h2 { font-size: 24px; }
h3 { font-size: 20px; }
h4 { font-size: 18px; }
p { font-size: 16px; }

@media (min-width: 768px) {
  h1 { font-size: 36px; } /* Desktop */
  h2 { font-size: 30px; }
  h3 { font-size: 24px; }
  h4 { font-size: 20px; }
}
```

**Readability:**
- Line length: 60-80 characters (mobile)
- Line height: 1.5-1.6
- Contrast ratio: 4.5:1 minimum (WCAG AA)

**C. TOUCH TARGETS:**

**Minimum Sizes:**
```css
.button, .link, .tap-target {
  min-height: 44px; /* iOS recommendation */
  min-width: 44px;
  padding: 12px 20px;
}

.touch-spacing {
  margin: 8px; /* Space between tap targets */
}
```

**Requirements:**
- Buttons: 44x44px minimum (48x48px ideal)
- Links: Adequate spacing (8px minimum)
- Form inputs: 44px height minimum
- Checkbox/radio: 24x24px minimum (with 44px tap area)

**D. PERFORMANCE (CRITICAL):**

**Core Web Vitals Targets:**
```
LCP (Largest Contentful Paint): < 2.5s ‚úÖ GOOD
FID (First Input Delay): < 100ms ‚úÖ GOOD
CLS (Cumulative Layout Shift): < 0.1 ‚úÖ GOOD
```

**Optimization Strategies:**

**Images:**
```html
<!-- Use WebP with fallback -->
<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Description" loading="lazy">
</picture>

<!-- Or modern approach -->
<img src="image.jpg" 
     srcset="image-320w.jpg 320w,
             image-480w.jpg 480w,
             image-800w.jpg 800w"
     sizes="(max-width: 320px) 280px,
            (max-width: 480px) 440px,
            800px"
     alt="Description"
     loading="lazy"
     width="800"
     height="600">
```

**Image Requirements:**
- WebP format (with JPG fallback)
- Responsive images (srcset)
- Lazy loading (except above fold)
- Proper dimensions specified (prevents CLS)
- Compressed (<200KB hero, <100KB others)
- CDN delivery

**CSS Optimization:**
```html
<!-- Critical CSS inline -->
<style>
  /* Inline critical above-the-fold CSS */
</style>

<!-- Non-critical CSS defer -->
<link rel="preload" href="styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="styles.css"></noscript>
```

**JavaScript Optimization:**
```html
<!-- Defer non-critical JS -->
<script src="script.js" defer></script>

<!-- Async for independent scripts -->
<script src="analytics.js" async></script>
```

**Font Loading:**
```html
<!-- Preload critical fonts -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

<!-- Font display swap -->
<style>
  @font-face {
    font-family: 'YourFont';
    src: url('font.woff2') format('woff2');
    font-display: swap; /* Prevents invisible text */
  }
</style>
```

**E. LAYOUT STABILITY (CLS Prevention):**

**Prevent Layout Shift:**
```html
<!-- Specify image dimensions -->
<img src="image.jpg" width="800" height="600" alt="...">

<!-- Reserve space for ads -->
<div class="ad-container" style="min-height: 250px;">
  <!-- Ad loads here -->
</div>

<!-- Font loading strategy -->
<style>
  body {
    font-family: system-ui, -apple-system, sans-serif; /* Fallback */
  }
  
  body.fonts-loaded {
    font-family: 'YourFont', system-ui, sans-serif;
  }
</style>
```

**Avoid:**
- Inserting content above existing content
- Ads without reserved space
- Embeds without dimensions
- Web fonts without fallback
